Site Features

  • Site desenvolvido por três alunos graduandos na disciplina de Desenvolvimento de Sistemas na WEB pela Universidade Federal de Itajubá.
  • As tecnologias utilizadas para o projeto foram Angular, Node e mySQL.
  • As iformações referentes aos produtos, clientes e pedidos foram armazenadas utilizando um banco de dados mySQL.
  • Banco de dados mySQL
    • Normalizada para a Terceira forma normal. As tables incluem:
      • descrição dos livros
      • categoria dos livros
      • relacionamento descrição-livros (vários para vários)
      • autores
      • relacionamento autor-livros (vários para vários)
      • clientes
      • pedidos
      • itens pedidos (um para vários)
  • Home page
    • Seleciona de forma aleatória três itens do banco de dados, utilizando comandos SQL.
    • Gera o menu de busca dinamicamente a partir do banco de dados usando uma query SQL. Se não houver livros na categoria exibe uma mensagem.
    • Limita e padroniza a descrição dos livros em até 200 caracteres.
  • Mecanismo de busca
    • Limpa os dados preenchidos por usuários para sua proteção contra ataques por injeções SQL e cross-site scripting.
    • As buscas são feitas pelo título do livro, descrição, autor e campos de categorias no banco de dados.
    • A função mysql_num_rows() é utilizada para contar o número de livros encontrados pela busca.
    • Responde por buscas que retornam nenhum resultado.
  • Página de Checkout
    • Procura pelos emails existentes no banco de dados as contas dos clientes e escreve nas informações de envio na forma da ordem de confirmação.
  • Página de confirmação de pedido
    • Checa por carrinhos de compra e avisa o usuário caso o carrinho esteja vazio.
    • Todos os campos são verificados para assegurar que há informação neles.
    • Procura pelos emails existentes no banco de dados e pede ao usuário para tentar novamente se o endereço não for encontrado.
    • Modificações feitas sobre as informações do consumidor são atualizadas no banco.
    • As informações do pedido são salvas no banco.
  • Histórico de pedidos
    • Procura no banco todos os pedidos associados com o ID do cliente.
    • Se nenhum resultado for encontrado haverá um aviso.
