Site Features

  • Site desenvolvido por três alunos graduandos na disciplina de Desenvolvimento de Sistemas na WEB pela Universidade Federal de Itajubá.
  • As tecnologias utilizadas para o projeto foram Angular, Node e mySQL.
  • As iformações referentes aos produtos, clientes e pedidos foram armazenadas utilizando uma base de dados mySQL.
  • Base de dados mySQL
    • Normalizada para a Terceira forma normal. As tables incluem:
      • descrição dos livros
      • categoria dos livros
      • relacionamento descrição-livros (vários para vários)
      • autores
      • relacionamento autor-livros (vários para vários)
      • clientes
      • pedidos
      • itens pedidos (um para vários)
  • Home page
    • Seleciona de forma aleatória três itens da base de dados, utilizando comandos SQL.
    • Gera o menu de busca dinamicamente a partir da base de dados usando uma query SQL que mostra somente as categorias que realmente contêm livros.
    • Limita e padroniza a descrição dos livros em até 250 caracteres.
  • Mecanismo de busca
    • Limpa os dados preenchidos por usuários para sua proteção contra ataques por injeções SQL e cross-site scripting.
    • As buscas são feitas pelo título do livro, descrição, autor e campos de categorias na base de dados.
    • A função mysql_num_rows() é utilizada para contar o número de livros encontrados pela busca.
    • Responde educadamente por buscas que retornam nenhum resultado.
  • Shopping cart page
    • Uses a cookie to store the ISBNs of items in the cart.
  • Página de Checkout
    • Procura pelos emails existentes na base de dados as contas dos clientes e escreve nas informações de envio na forma da ordem de confirmação.
    • O ID do usuário é criptografado utilizando o algoritmo Rijndael
  • Página de confirmação de pedido
    • Checa por carrinhos de compra e avisa o usuário caso o carrinho esteja vazio.
    • Todos os campos são verificados para assegurar que há informação neles.
    • Procura pelos emails existentes na base de dados e pede ao usuário para tentar novamente se o endereço não for encontrado.
    • Modifications made to custo mer information are updated in the database.
    • Order information are written to the database.
    • An email is sent to the customer with the order information.
    • The shopping cart is emptied by setting ItemCount to zero in the ShoppingCart cookie.
  • Order History Page
    • Searches the database for all orders associated with e-mail address
    • If no matching email address is found user is prompted to try again.
